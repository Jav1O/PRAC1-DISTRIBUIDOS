                               Ejercicio Evaluable 1: colas de mensajes

Considere una aplicación monolítica con la siguiente estructura:




El archivo claves.c implementa un servicio para almacenar tuplas de la forma <key-value1-
value2-value3>, donde:

    -   key es la clave que será una cadena de caracteres de como mucho 255 caracteres (excluido el
        código 0 que permite indicar el fin de la cadena).
    -   Los valores asociados a la clave serán los siguientes:
            o value1: una cadena de caracteres de como mucho 255 caracteres (excluido el código 0
                que permite indicar el fin de la cadena).
            o value2: un vector de N elementos de tipo float (coma flotante de simple precisión). El
                valor 2 vendrá dado por dos elementos: un número N que determina la longitud del
                vector, y el vector con los elementos. El valor de N será cualquier número comprendido
                entre 1 y 32.
            o value3: una estructura de datos que almacena tres valores enteros (x, y, z). Esta
                estructura se define de la siguiente forma:

                       struct Paquete {
                             int x;
                             int y;
                             int z;
                       };

    La API (Application Progamming Interface) de este servicio, cuyo prototipo en C se encuentra
    declarado en el archivo claves.h, incluye las siguientes funciones:

   •    int destroy(void). Esta llamada permite inicializar el servicio de elementos key-value1-
        value2-value3. Mediante este servicio se destruyen todas las tuplas que estuvieran
        almacenadas previamente. La función devuelve 0 en caso de éxito y -1 en caso de error.
   •    int set_value(char *key, char *value1, int N_value2, float *V_value2,
        struct Paquete value3). Este servicio inserta el elemento <key, value1, value2,
                                                                                                    1
       value3>. El vector correspondiente al valor 2 vendrá dado por la dimensión del vector
       (N_value2) y el vector en sí (V_value2). El servicio devuelve 0 si se insertó con éxito y -1 en
       caso de error. Se considera error intentar insertar una clave que ya existe previamente, o que el
       valor N_value2 esté fuera de rango (los valores tienen que estar comprendidos entre 1 y 32).
       En estos casos se devolverá -1 y no se insertará.
   •   int get_value(char *key, char *value1, int *N_value2, float *V_value2,
       struct Paquete *value3). Este servicio permite obtener los valores asociados a la clave
       key. La cadena de caracteres asociada se devuelve en value1. En N_value2 se devuelve la
       dimensión del vector asociado al valor 2 y en V_value2 los componentes del vector. En value3
       se devuelve la estructura asociada. Tanto value1 como V_value2 tienen que tener espacio
       reservado para poder almacenar el máximo número de elementos posible (255 en el caso de la
       cadena de caracteres excluido el código de fin de cadena, y 32 en el caso del vector de floats).
       La función devuelve 0 en caso de éxito y -1 en caso de error, por ejemplo, si no existe un elemento
       con dicha clave.
   •   int modify_value(char *key, char *value1, int N_value2, float *V_value2,
       struct PAquete value3). Este servicio permite modificar los valores asociados a la clave
       key. La función devuelve 0 en caso de éxito y -1 en caso de error, por ejemplo, si no existe un
       elemento con dicha clave. También se devolverá -1 si el valor N_value2 está fuera de rango.
   •   int delete_key(char *key). Este servicio permite borrar el elemento cuya clave es key. La
       función devuelve 0 en caso de éxito y -1 en caso de error. En caso de que la clave no exista
       también se devuelve -1.
   •   int exist(char *key). Este servicio permite determinar si existe un elemento con clave key.
       La función devuelve 1 en caso de que exista y 0 en caso de que no exista.


Todas las operaciones anteriores deben realizarse de forma atómica.

A continuación, se muestra un archivo, denominado app_cliente.c. Este archivo es un ejemplo
de aplicación que puede hacer uso de esta API. Un ejemplo posible sería el siguiente:

       #include <stdio.h>
       #include "claves.h"

       int main (int argc, char **argv)
       {
             char *key = "clave 5";
             char *v1 = "ejemplo de valor 1";
             float v2[] = {2.3, 0.5, 23.45};
             struct Paquete v3;

               v3.x = 10;
               v3.y = 5;
               v3.z = 3;

               int err = set_value(key, v1, 3, v2, v3);
               if (err == -1) {
                   printf("Error al insertar la tupla\n");
               }

               return 0;
       }

En esta versión se han de llevar a cabo las siguientes tareas:

    1. Desarrollar el código (claves.c) que gestiona el almacenamiento y procesamiento de las tuplas.

    4. Elaborar un plan de pruebas de la biblioteca desarrollada. Este plan se probará con el código

B.- Desarrollar una versión distribuida utilizando colas de mensajes. En este caso, hay que diseñar e
implementar, utilizando exclusivamente colas de mensajes POSIX y el lenguaje de programación C, el
sistema que implementa el servicio de tuplas key-value1-value2-value3, con la estructura
mostrada en la Figura 2. En este ejercicio se trata de convertir la aplicación desarrollada en el apartado
A en un servicio distribuido de forma que exista un servidor que gestione las claves y un cliente que tenga
acceso a la API anterior, pero de forma distribuida. Para ello debe:

    1. Desarrollar el código del servidor (servidor-mq.c) encargado de las comunicaciones con la
       parte cliente. Este servidor debe ser concurrente, recibirá las peticiones del lado cliente e
       invocará a las funciones de la biblioteca dinámica libclaves.so implementadas en el apartado A.

En esta aplicación los ficheros claves.h, claves.c y app-cliente.c son los mismos que se tendrían
en una aplicación monolítica. La interfaz de claves.c es idéntica a la anterior. Como en este caso existe
una comunicación entre la parte cliente y la parte servidora, las funciones anteriores devolverán -1
cuando hay un error relacionado con el servicio de tuplas y -2 cuando hay un error producido por el
servicio de comunicaciones. Es decir, las llamadas anteriores devolverán -2 cuando se produzca un error
en el sistema de paso de mensajes (colas inexistentes, errores al enviar datos a una cola, errores al recibir
datos de las colas, etc.). También se considerará error y se devolverá -1 sin necesidad de contactar con el
servidor, en caso de que a las llamadas anteriores se pasen cadenas de más de 255 caracteres para el
valor value1 o cuando el valor N_value2 sea mayor de 32.

Para el almacenamiento de los elementos key-value1-value2-value3 puede hacer uso de la
estructura de datos o mecanismo de almacenamiento que considere más adecuado (listas, ficheros, etc.),
el cual describirá en la memoria entregada. La estructura elegida no debe fijar un límite en el número de
elementos que se pueden almacenar, ni debe suponer instalar software adicional (por ejemplo, una base
de datos).

Se recomienda probar el servidor con varios clientes de forma concurrente.

Tenga en cuenta que el prototipo de las funciones (claves.h) destroy, set_value, get_value,
delete_key, modify_value y exist no puede ser modificado. En Aula global ya se proporciona
el archivo claves.h.

Aclaraciones adicionales:

Cuando se esté desarrollando este ejercicio, es posible que en algunas ocasiones las colas de mensajes
se queden en un estado inconsistente y tanto el cliente como el servidor fallen en tiempo de ejecución.
En estos casos, lo mejor es borrar las colas de mensajes que se hayan quedado en el sistema. Para ello,
pueden eliminarse las colas desde el directorio /dev/mqueue.
